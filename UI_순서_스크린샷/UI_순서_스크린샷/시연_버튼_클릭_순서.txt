시연 중 자연스럽게 말할 포인트

1. 클라이언트는 처음부터 업체를 고르는 것이 아니라, 먼저 도면과 조건만 넣는다.
2. AI가 누락 정보와 위험 요소를 잡아주고, 클라이언트가 필요한 부분만 보완한다.
3. 관리자는 모든 일을 직접 처리하기보다, 요청/매칭/계약/생산/납품/리뷰 상태가 정상적으로 흘러가는지 관제한다.
4. 가공업체는 오더리스트를 보고 가능한 건만 수락하고, 견적과 납기를 회신한다.
5. 클라이언트는 업체별 가격, 납기, 평점, 대응 가능 공정을 비교해 최종 업체를 선택한다.
6. 계약/NDA/결제대행은 발주 이후 중요한 거래 보호 장치로 설명한다.
7. 생산 이후에는 업체가 이벤트를 업데이트하고, 클라이언트와 관리자가 각각 필요한 화면에서 확인한다.
8. 납품 후 클라이언트가 인증하고 리뷰를 남기면 관리자 DB에 완료 거래로 쌓인다.


시연 중 피해야 할 설명

1. 관리자가 직접 제조나 송금을 수행한다고 말하지 않기
   -> 관리자는 상태 확인, 중재, 로그 관리, 정산 조건 확인 역할로 설명

2. 업체가 클라이언트를 평가한다고 말하지 않기
   -> 이번 시연은 클라이언트가 업체 리뷰를 남기고, 업체는 받은 리뷰를 확인하는 흐름

3. 견적 요청 등록 화면에서 바로 계약이 확정됐다고 말하지 않기
   -> 견적 요청은 RFQ 생성 단계이고, 계약은 업체 선택 후 발주 단계에서 진행

4. 모든 버튼이 실제 서버 API를 호출한다고 말하지 않기
   -> 현재는 프론트 시연용 UI이며, 흐름과 화면 전환을 보여주는 목적
